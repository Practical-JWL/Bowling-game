🎳 볼링 300 시뮬레이터: 코드 로직 및 구조 정리
본 문서는 **Flet(Python UI Framework)**을 활용하여 제작된 멀티플레이어 볼링 시뮬레이터의 핵심 로직과 코드 실행 순서를 정리한 가이드입니다.

1. 볼링 게임 핵심 규칙 (Domain Logic)
코드 내에 구현된 볼링 점수 계산 및 진행 규칙입니다.

📋 프레임 및 투구 규칙
기본 구성: 총 10프레임으로 진행됩니다.

일반 프레임 (1~9프레임):

최대 2번의 투구 기회가 주어집니다.

스트라이크(X): 첫 투구에 10핀을 모두 쓰러뜨리면 해당 프레임 종료.

스페어(/): 두 번째 투구까지 합쳐서 10핀을 쓰러뜨린 경우.

마지막 프레임 (10프레임):

스트라이크나 스페어를 기록하면 최대 3번까지 투구가 가능합니다.

🔢 점수 계산 로직 (calculate_total_score)
볼링은 투구 시점보다 나중에 점수가 확정되는 '누적 합산' 방식입니다.

스트라이크: 10점 + 다음 2번의 투구 점수 합산.

스페어: 10점 + 다음 1번의 투구 점수 합산.

오픈: 스트라이크/스페어 실패 시 해당 프레임에서 쓰러뜨린 핀 수만 합산.
